## 🚀 시작하기

### 최초 실행

1. **Code-tare.app** 더블클릭으로 실행
2. 처음 실행하면 기본 설정으로 `~/code_workshop` 폴더를 스캔합니다
3. 설정 메뉴에서 원하는 프로젝트 폴더로 변경 가능

### 빠른 설정

- **⚙️ 설정** 메뉴 클릭
- **프로젝트 부모 폴더** 선택
- 저장하면 자동으로 프로젝트 스캔 시작

## ✨ 주요 기능

### 1. 자동 프로젝트 스캔

Code-tare는 지정된 폴더의 하위 디렉토리를 자동으로 스캔하여:

- **프로젝트 타입 감지**: Node.js, Python, Rust, Go 등
- **기술 스택 분석**: package.json, requirements.txt, Cargo.toml 등
- **Git 원격 저장소**: GitHub URL 자동 추출
- **README 설명**: 자동으로 프로젝트 설명 추출
- **최종 수정일**: 파일 시스템 기준 정렬

### 2. 태그 시스템

프로젝트를 체계적으로 관리:

- **진행 상태**: 진행중, 중지, 완료, 계획중, deprecated
- **구분 태그**: 사이드 프로젝트, 업무, 학습, 오픈소스 등 (커스텀 가능)
- **즐겨찾기**: ⭐ 중요한 프로젝트 표시
- **아카이브**: 📦 완료된 프로젝트 정리
- **커스텀 제목**: 프로젝트 폴더명 대신 표시할 이름 설정

### 3. 원클릭 실행

각 프로젝트 카드에서 버튼 클릭:

- **🤖 Claude**: 터미널 앱에서 프로젝트 열기
- **💻 VS Code**: 에디터로 프로젝트 열기
- **📁 Finder**: 파일 탐색기로 폴더 열기
- **🔗 GitHub**: 웹 브라우저로 저장소 열기

```

│                                 │

│                                 │
│ [🤖] [💻] [📁] [🔗]            │
└─────────────────────────────────┘
```

**검색바**:
- 프로젝트 이름, 설명, 기술 스택으로 검색
- 실시간 필터링

**좌측 필터 패널**:
- 진행 상태별 필터
- 구분 태그별 필터
- 즐겨찾기만 보기
- 여러 필터 동시 적용 가능

### 정렬

- **기본**: 최종 수정일 내림차순 (최신순)
- 자동으로 최근 작업한 프로젝트가 상단에 표시

## ⚙️ 설정 가이드

### 프로젝트 부모 폴더

```
예시:
/Users/your-name/Projects
├── my-website/          ✅ 스캔됨
├── python-scripts/      ✅ 스캔됨
├── rust-cli/            ✅ 스캔됨
└── node_modules/        ❌ 제외됨
```

**설정 방법**:
1. ⚙️ 설정 메뉴
2. "프로젝트 부모 폴더" → "폴더 선택" 버튼
3. 프로젝트들이 있는 **상위 폴더** 선택
4. 저장

### 터미널 앱 설정

**기본 제공**:
- Warp (기본값)
- iTerm
- Terminal
- Alacritty
- Kitty

**커스텀 설정**:
1. 드롭다운에서 "기타 (직접 입력)" 선택
2. macOS 애플리케이션 이름 입력
   - 예: "Hyper", "Tabby"

### 에디터 커맨드 설정

**기본 제공**:
- `code` (VS Code)
- `cursor`
- `zed`
- `subl` (Sublime Text)
- `atom`
- `nvim`

**커스텀 설정**:
1. 드롭다운에서 "기타 (직접 입력)" 선택
2. PATH에 등록된 커맨드 입력
   - 예: `vim`, `emacs`, `idea`

### 스캔 제외 폴더

프로젝트 스캔 시 건너뛸 폴더 이름:

**기본 제외 목록**:
```
node_modules, .git, .cache, target, dist, build,
.next, .nuxt, venv, __pycache__, .venv
```

**추가/제거**:
- 폴더 이름 입력 → "추가" 버튼
- 태그 클릭 → "×" 버튼으로 제거

### 표시 옵션

- ✅ **아카이브된 프로젝트 숨기기**: 완료된 프로젝트 표시 안 함
- ✅ **숨김 프로젝트 가리기**: `_` 또는 `.`으로 시작하는 폴더 제외

## 🏷️ 태그 관리

### 프로젝트별 태그 설정

1. 프로젝트 카드에서 **"태그"** 버튼 클릭
2. 모달 창에서 설정:
   - **커스텀 제목**: 표시할 이름 변경
   - **진행 상태**: 단일 선택
   - **구분 태그**: 다중 선택 가능
   - **즐겨찾기**: ⭐ 토글
   - **아카이브**: 📦 토글
   - **메모**: 자유 텍스트

### 전역 태그 관리

**🏷️ 태그 관리** 메뉴에서:

1. **구분 태그 추가**:
   - 예: "클라이언트 프로젝트", "실험", "포트폴리오"
   - 자동으로 색상 배정

2. **태그 삭제**:
   - 사용 중이지 않은 태그만 삭제 가능
   - 프로젝트에 할당된 태그는 삭제 불가

3. **색상 자동 배정**:
   - 태그마다 고유한 색상 표시
   - 시각적으로 구분 용이

## 💡 팁 & 트릭

### 프로젝트 자동 스캔 설정

특정 파일이 있으면 프로젝트로 인식:
- `package.json` → Node.js
- `Cargo.toml` → Rust
- `go.mod` → Go
- `requirements.txt` → Python
- `.git` 폴더 → Git 저장소

### 효율적인 태그 활용

1. **진행 상태**로 작업 우선순위 관리
   - "진행중" 필터 → 현재 작업 중인 것만 보기

2. **구분 태그**로 프로젝트 분류
   - "업무" 태그 → 회사 프로젝트
   - "사이드 프로젝트" → 개인 작업

3. **즐겨찾기**로 자주 쓰는 것 표시
   - ⭐ 필터 → 주요 프로젝트만 빠르게 접근

4. **아카이브**로 정리
   - 완료된 프로젝트 숨기기
   - 필요할 때만 표시

### 검색 활용

- **프로젝트 이름**: `my-app`
- **기술 스택**: `React` → React 사용하는 모든 프로젝트
- **설명**: `API` → 설명에 "API" 포함된 프로젝트

## ❓ 자주 묻는 질문 (FAQ)

### Q: 프로젝트가 스캔되지 않아요

**A**: 다음을 확인하세요:
1. 설정에서 프로젝트 부모 폴더가 올바른지 확인
2. 폴더 이름이 제외 목록에 있는지 확인 (`_`, `.` 시작)
3. 프로젝트 폴더에 인식 가능한 파일이 있는지 확인
   - `package.json`, `Cargo.toml`, `.git` 등

### Q: 터미널이 제대로 열리지 않아요

**A**:
1. 설정에서 터미널 앱 이름이 정확한지 확인
2. macOS 애플리케이션 이름과 일치해야 함
3. 예: "Warp" (O), "Warp.app" (X)

### Q: 에디터 명령어가 작동하지 않아요

**A**:
1. 에디터가 PATH에 등록되어 있는지 확인
2. 터미널에서 `which code` 실행해서 확인
3. VS Code의 경우: `Cmd+Shift+P` → "Shell Command: Install 'code' command in PATH"

### Q: README가 표시되지 않아요

**A**:
- 프로젝트 루트에 `README.md` 파일이 있어야 합니다
- 앱을 재시작해보세요

### Q: 태그 색상을 변경할 수 있나요?

**A**:
- 현재 버전에서는 자동 색상 배정만 지원합니다
- 향후 업데이트에서 커스텀 색상 지원 예정

## 🆘 문제 해결

### 앱이 실행되지 않아요

1. macOS 보안 설정 확인
   - 시스템 설정 → 보안 및 개인정보 보호
   - "다운로드한 응용 프로그램 허용" 확인

2. 권한 문제
   - 터미널에서 실행: `chmod +x Code-tare.app/Contents/MacOS/*`

### 데이터가 사라졌어요

**데이터 저장 위치**:
```
~/Library/Application Support/com.tare.code-tare/
├── data/
│   ├── tags.json          # 태그 데이터
│   ├── colors.json        # 색상 설정
│   └── settings.json      # 앱 설정
```

백업 권장!
